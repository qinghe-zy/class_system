# 数据库设计说明

## 1. 设计原则

- 统一使用 `utf8mb4`
- 主键使用 `BIGINT AUTO_INCREMENT`
- 核心业务表保留 `created_time`、`updated_time`、`deleted_flag`
- 保持字段语义清晰，满足三端权限与统计口径
- 为高频查询字段建立索引

## 2. 核心数据表

1. `sys_user` 用户表
2. `sys_role` 角色表
3. `sys_user_role` 用户角色关联表
4. `sys_permission` 权限表（简化RBAC）
5. `course` 课程表
6. `course_audit_record` 课程审核记录表
7. `course_enrollment` 学生选课表
8. `class_session` 课堂会话表
9. `sign_in_task` 签到任务表
10. `sign_in_record` 签到记录表
11. `quiz` 测验表
12. `quiz_question` 测验题目表
13. `quiz_submission` 测验提交表
14. `behavior_raw_data` 行为原始数据表
15. `behavior_record` 行为检测结果表
16. `operation_log` 操作日志表

## 3. 关键关系

- 用户与角色：`sys_user` 1..n `sys_user_role` n..1 `sys_role`
- 课程与教师：`course.teacher_id -> sys_user.id`
- 选课关系：`course_enrollment(course_id, student_id)`
- 课堂会话：`class_session.course_id -> course.id`
- 签到/测验/行为检测均关联 `class_session`
- 行为原始数据与行为检测结果分离，支持复盘与算法扩展

## 4. 索引设计

- 用户：`idx_user_status`、`idx_user_real_name`
- 课程：`idx_course_teacher`、`idx_course_audit_status`
- 选课：`uk_course_student`
- 会话：`idx_session_course`、`idx_session_teacher`
- 行为检测：
  - `idx_behavior_record_teacher`
  - `idx_behavior_record_course`
  - `idx_behavior_record_detect_time`
  - `idx_behavior_record_exception`

## 5. 初始化数据

- 1个管理员
- 2个教师
- 5个学生
- 3门课程
- 课堂会话、签到、测验、行为检测演示数据已覆盖统计场景
