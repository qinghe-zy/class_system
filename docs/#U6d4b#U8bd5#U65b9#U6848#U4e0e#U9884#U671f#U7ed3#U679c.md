# 测试方案与预期结果

## 1. 测试目标

- 验证系统可运行、可联调、可展示。
- 验证三端权限隔离正确。
- 验证核心链路“课堂行为实时检测”闭环可用。
- 验证新增修复项：签到惩罚、课件附件、测验同步、填空友好性。

## 2. 测试范围

- 环境与依赖检查
- 数据库初始化与迁移
- 后端接口正确性
- 前端页面交互与联调
- 权限与数据权限控制
- 统计图表与数据口径一致性

## 3. 测试环境与前置数据

### 3.1 软件环境

- JDK 21+
- Maven 3.9+
- Node.js 18+
- npm 9+
- MySQL 8.0+

### 3.2 初始化步骤

```bash

mysql -uroot -p123456 < sql/data.sql
mysql --force -uroot -p123456 < sql/migration_20260318_force.sql
```

### 3.3 启动步骤

```bash
cd backend

```

```bash

npm run dev
```

### 3.4 演示账号

- 管理员：`admin / 123456`
- 教师：`teacher01 / 123456`

## 4. 测试方法

### 4.1 测试类型

- 接口测试：Postman / curl / Knife4j。
- 页面测试：浏览器手工点击验证。
- 数据验证：MySQL 查询落库结果。
- 逻辑验证：时间窗口、权限边界、状态流转。

### 4.2 结果判定标准

- `code = 0` 为成功。
- 失败分支必须返回明确错误信息。
- 关键业务需验证数据库真实变更。
