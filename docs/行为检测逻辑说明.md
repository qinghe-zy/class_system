# 行为检测逻辑说明

## 1. 模块分层

行为检测采用独立服务分层：

- `BehaviorCollectService`：接收前端心跳上报，保存原始数据
- `BehaviorAnalyzeService`：调用规则分析，生成判定结果
- `BehaviorRuleEngine`：规则引擎核心，计算活跃度与状态标签
- `BehaviorRecordService`：持久化检测结果

## 2. 前端采集指标

- `onlineFlag` 是否在线
- `lastActiveTime` 最近活跃时间
- `heartbeatTime` 心跳时间
- `pageStayDurationSec` 页面停留时长
- `focusFlag` 聚焦状态
- `blurDurationSec` 失焦时长
- `mouseClickCount` 点击次数
- `keyInputCount` 键盘输入次数
- `signInFlag` 签到状态
- `quizJoinFlag` 测验参与状态
- `classStayDurationSec` 课堂停留时长
- `baseActivityScore` 基础活跃分

## 3. 活跃度计算

```text
activityScore = base
  + mouseClickCount / 2
  + keyInputCount / 2
  - blurDurationSec / 10
  - (onlineFlag == 0 ? 30 : 0)
```

最终分值限制在 `[0, 100]`。

## 4. 规则判定

- 未签到：`signInFlag == 0`
- 轻度分心：`focusFlag == 0 && blurDurationSec >= 180`
- 长时间无操作：`mouse + key == 0 && pageStayDurationSec >= 300`
- 疑似挂机：`onlineFlag == 0` 或长时低交互
- 互动不足：`quizJoinFlag == 0 && activityScore < 40`

状态输出：
- `正常学习`
- `轻度分心`
- `长时间无操作`
- `疑似挂机`
- `互动不足`
- `未签到`
- 多项叠加时输出 `学习状态异常`

## 5. 落库策略

每次上报都执行两次落库：

1. 原始数据入 `behavior_raw_data`
2. 分析结果入 `behavior_record`

核心字段：
- `behavior_status`
- `behavior_type`
- `status_description`
- `activity_score`
- `exception_flag`
- `raw_data_summary`

## 6. 扩展点预留

- 机器学习模型：可替换 `BehaviorRuleEngine` 的输出来源
- 视频识别：可在 `BehaviorCollectService` 增加视频特征输入
- 第三方AI服务：可新增 `BehaviorExternalAnalyzeService`
- 预警通知：可在 `BehaviorRecordService` 后置事件触发告警
